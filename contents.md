# エンジニアが雰囲気で使ってるIT用語を正しく学ぶ本の企画案(改訂版)

## 書籍コンセプト

**タイトル案:**

- 「それ、本当に合ってます? エンジニアの”なんとなく”IT用語辞典」
- 「プルリク出す前に読む本〜雰囲気で使ってたあの言葉〜」
- 「技術書には載ってない、現場のIT用語のモヤモヤ全部解消」

**ターゲット読者:**

- 実務経験1〜5年のエンジニア
- 資格は持ってるけど現場用語に戸惑う人
- 「分かったフリ」で乗り切ってきた中堅層

-----

## 章立て構成

### 第1章:「Gitで雰囲気で使ってる」系

**特徴:** 毎日使うのに実は曖昧な用語

- **マージとリベース** → 「結果同じじゃね?」からの深い違い
- **フェッチとプル** → 混同しがちな兄弟
- **チェリーピック** → さくらんぼ摘み?コミット摘み?
- **スカッシュ** → 押しつぶすって何を?
- **ステージング** → add したら staging area に…って?
- **HEAD、origin、upstream** → 誰が誰やら問題
- **fast-forward** → 早送り?マージの一種?

**ウケる仕掛け:** 「Git操作、本当は何が起きてるの?」図解

### 第2章:「開発フローで雰囲気で使ってる」系

**特徴:** スクラムとかアジャイルとか

- **スプリント** → なぜ短距離走?
- **バックログ** → 溜まってる仕事…だけ?
- **リファクタリング** → リライトとの違いは?
- **テクニカルデット** → 技術的負債、具体的には?
- **ポストモーテム** → 死後?振り返り?
- **インクリメント** → 増分?成果物?
- **ベロシティ** → 速度を測る意味とは

**ウケる仕掛け:** 「先輩の使い方、実は間違ってた」事例集

### 第3章:「ビジネス会議で雰囲気で使ってる」系

**特徴:** エンジニアが苦手な経営・ビジネス用語

- **ROI** → 投資対効果…技術選定でどう使う?
- **KPI、KGI、OKR** → 全部目標っぽいけど違いは?
- **ステークホルダー** → 利害関係者…誰が該当?
- **アライアンス** → 提携?パートナーシップとの違いは?
- **リソース** → 人?お金?時間?サーバー?
- **フィージビリティ** → 実現可能性…スタディって?
- **マイルストーン** → 石の塚?中間目標?
- **ボトムアップ vs トップダウン** → 開発手法じゃないの?
- **コンセンサス** → 合意形成…全員一致?
- **タスクフォース** → 特別チーム…普通のチームとの違いは?
- **エスカレーション** → 上申?どのタイミングで?
- **グロースハック** → 成長…ハック?
- **ピボット** → 方向転換…仕様変更とは違う?
- **MVP(Minimum Viable Product)** → 最小限…どこまで削る?
- **PMF(Product Market Fit)** → 市場適合…エンジニアに関係ある?
- **バリュープロポジション** → 価値提案…機能一覧じゃダメ?
- **バーンダウンチャート** → 燃え尽き…グラフ?
- **ウォーターフォール** → 滝?古い開発手法だけ?
- **コアコンピタンス** → 中核能力…強みと違う?
- **シナジー効果** → 相乗効果…具体的には?

**ウケる仕掛け:**

- 「経営会議で生き残るエンジニアの護身術」
- 「ビジネス用語→エンジニア用語 翻訳表」
- 「『技術的には可能ですが…』の後に続く言葉選び」

**コラム:**

- 「エンジニアとビジネス職の『優先度』の認識ズレ問題」
- 「『工数見積もり』と『ビジネス価値』の翻訳ガイド」
- 「技術負債をビジネス用語で説明する方法」

### 第4章:「アーキテクチャで雰囲気で使ってる」系

**特徴:** 設計の話で出てくる謎ワード

- **疎結合と密結合** → どっちがいいの?
- **ステートレスとステートフル** → 状態を持つ/持たないの何が違う?
- **イミュータブル** → 不変…だから何?
- **ポリモーフィズム** → 多態性って言われても
- **デプリケート** → 非推奨と廃止の違い
- **ペイロード** → 荷物?データ本体?
- **エンドポイント** → 終点?いやAPIの…
- **スロットリング** → 絞る?制限?

**ウケる仕掛け:** 「設計レビューで恥をかかないための用語チェックリスト」

### 第5章:「インフラ・ネットワークで雰囲気で使ってる」系

**特徴:** SREとかDevOpsとか

- **オーケストレーション** → コンテナの指揮者?
- **プロビジョニング** → 準備?設定?構築?
- **イングレスとエグレス** → 入口と出口、だけ?
- **ロードバランサー** → 負荷分散は分かるけど仕組みは?
- **リバースプロキシ** → 何が逆なの?
- **サイドカー** → バイクの横?コンテナの話?
- **メッシュネットワーク** → 網目状…で?
- **サーキットブレーカー** → ブレーカー落とす?

**ウケる仕掛け:** 「インフラ担当との会話で困らない用語マップ」

### 第6章:「データベースで雰囲気で使ってる」系

**特徴:** SQL書けるけど説明できない

- **トランザクション** → 処理のかたまり…?
- **ACID特性** → 酸性じゃないよ
- **正規化** → 第1、第2、第3…覚えてます?
- **インデックス** → 速くなるやつ、以上
- **シャーディング** → 分割?パーティショニングとの違いは?
- **レプリケーション** → 複製してるのは知ってる
- **オプティミスティックロック** → 楽観的…って何が?
- **N+1問題** → Nって何やねん

**ウケる仕掛け:** 「DBチューニング会議で生き残る最低限の知識」

### 第7章:「セキュリティで雰囲気で使ってる」系

**特徴:** なんとなく怖いやつ

- **サニタイズとバリデーション** → 消毒と検証?
- **CSRF、XSS、SQLインジェクション** → 略語の元は?
- **ゼロトラスト** → 信用しない…誰を?
- **ソルト** → パスワードに塩?
- **ノンス** → 使い捨て番号…なぜ?
- **オリジン** → 起源?ドメインのこと?
- **CORS** → クロスオリジン…何?
- **ホワイトリスト/ブラックリスト** → 今は言っちゃダメ?

**ウケる仕掛け:** 「脆弱性診断レポートを読むための用語集」

### 第8章:「パフォーマンスで雰囲気で使ってる」系

**特徴:** 速い/遅いの話で出てくる謎指標

- **レイテンシとスループット** → 遅延と処理量…の違いは?
- **キャッシュヒット率** → 高い方がいいのは分かる
- **ボトルネック** → 瓶の首?
- **デグレード** → 劣化?デグレ?
- **メモリリーク** → 漏れてる…どこに?
- **GC(ガベージコレクション)** → ゴミ収集、いつ?
- **プロファイリング** → 犯罪捜査じゃないよ
- **ベンチマーク** → 測定してるのは何?

**ウケる仕掛け:** 「遅い!って言われたときの調査フローチャート」

### 第9章:「CI/CDで雰囲気で使ってる」系

**特徴:** パイプライン作ってるけど…

- **アーティファクト** → 成果物?どの段階の?
- **ブルーグリーンデプロイ** → 色の意味は?
- **カナリアリリース** → カナリア?炭鉱?
- **ロールバック** → 戻すのは分かる、どこまで?
- **イミュータブルインフラ** → 不変…壊して作る?
- **Feature Flag** → フラグ?トグル?
- **Smoke Test** → 煙?どういうテスト?
- **ヘルスチェック** → 何の健康診断?

**ウケる仕掛け:** 「デプロイ失敗時に慌てないための用語理解」

### 第10章:「コードレビューで雰囲気で使ってる」系

**特徴:** レビューコメントの謎ワード

- **LGTM** → Looks Good To Me、でも本当に?
- **IMO、IMHO** → 個人的意見…の略?
- **nit** → ニット?細かい指摘?
- **WIP** → Work In Progress、プルリクに書くけど
- **YAGNI** → You Aren’t Gonna Need It、いつ使う?
- **DRY** → Don’t Repeat Yourself、どこまで?
- **SOLID原則** → 頭文字…全部言える?
- **コードスメル** → 臭い?具体的には?

**ウケる仕掛け:** 「レビュアーの真意を読み解く」会話例

### 第11章:「モダン開発で雰囲気で使ってる」系

**特徴:** 最近よく聞くバズワード的な

- **Serverless** → サーバーないの?あるの?
- **Edge Computing** → エッジ…端っこ?
- **Observability** → 可観測性って何観測?
- **Gitops** → Gitで運用…?
- **IaC(Infrastructure as Code)** → コード化の本質は?
- **Service Mesh** → マイクロサービスの網?
- **Jamstack** → ジャムの山?
- **Monorepo vs Multirepo** → どっちがいいの論争

**ウケる仕掛け:** 「技術選定会議で知ったかぶりしないための最新用語」

## 特別付録

### 付録1: 「雰囲気使用度チェックシート」

50の用語について「説明できる/雰囲気で使ってる/知らない」を自己診断

### 付録2: 「現場の文脈別」用語索引

- Slackでよく見る用語
- プルリクでよく見る用語
- 障害対応でよく聞く用語
- 設計書でよく見る用語
- **経営会議でよく聞く用語(NEW!)**

### 付録3: 「英語の元の意味から理解する」一覧

カタカナ語の語源を知ることで本質理解

### 付録4: 「間違えやすい類似用語」対照表

migration vs upgrade、deprecate vs obsolete など

### 付録5: 「エンジニア⇔ビジネス 翻訳辞典(NEW!)」

- 技術用語をビジネス用語で説明する
- ビジネス要求を技術要件に翻訳する
